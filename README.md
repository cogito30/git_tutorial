# git_tutorial
미니 프로젝트를 통한 git 튜토리얼

## Index

#### 1. Git 생존 필수기
- 핵심개념: 버전관리의 필요성, Working Directory, Staging Area, Repository
- 명령어: `git init`, `git clone`, `git add`, `git commit`
- 포인트: .gitignore의 필요성, commit message 작성법

#### 2. 협업의 기본, 브랜치와 병합
- 핵심개념: 왜 main(master) 브랜치에 서 직접 작업하면 안되는지
- 명령어: `git branch`, `git switch`, `git merge`
- 포인트: conflict 해결하기

#### 3. 팀 단위 작업하기
- 핵심개념: Git Flow와 GitHub Flow 전략
- 명령어: `git fetch`, `git pull`
- 포인트: PR(Pull Request) 및 코드 리뷰

#### 4. 실수를 수습하는 방법, 복구 기술
- 핵심개념: Local에만 있는 commit, 원격저장소에 올라간 commit 복구 방법
- 명령어: `git commit --amend`, `git stash`, `git stash pop`, `git reset`, `git revert`, `git reflog`
- 포인트: 날린 브랜치나 코드 기록을 복구

#### 5. 심화과정
- 핵심개념: 깔끔한 commit 히스토리 작성법
- 명령어: `git rebase -i`, `git cherry-pick`
- 포인트: merge와 rebase의 차이점과 상황에 따른 선택 기준 파악

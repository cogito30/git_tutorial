안녕하세요! 오늘부터 여러분의 '실무 Git' 생존 능력을 키워줄 튜토리얼을 시작합니다.

혹시 대학교 조별 과제나 개인 프로젝트를 하면서 이런 파일명 만들어 본 적 있으신가요?

```
📄 최종_보고서.docx
📄 최종_진짜_보고서.docx
📄 최종_진짜_마지막_보고서.docx
📄 최종_이게진짜끝_교수님살려주세요.docx
```

개발을 할 때도 코드를 이렇게 관리한다면 어떨까요? 파일은 수백 개고, 어제 잘 되던 코드가 오늘 갑자기 안 되는데 어디서부터 잘못된 건지 찾을 길이 없을 겁니다.
이 끔찍한 상황을 구원해 주는 도구가 바로 **`Git(깃)`**입니다.

## 1. Git과 GitHub, 똑같은 거 아니었어? 🤔

Git을 처음 접하는 분들이 가장 많이 헷갈려하는 부분입니다. 이름이 비슷해서 같은 건 줄 아는 분들이 많지만, 둘은 완전히 다른 역할을 합니다.
- Git (비디오 카메라 🎥): 내 컴퓨터(Local)에서 코드의 변경점(버전)을 기록하고 관리하는 프로그램입니다. 오프라인 상태에서도 작동합니다.
- GitHub (유튜브 📺): Git으로 기록한 코드(비디오)를 인터넷에 올려 다른 사람들과 공유하고 협업할 수 있게 해주는 웹 서비스(플랫폼)입니다.
즉, "Git을 사용해서 내 코드를 관리하고, 그 기록을 GitHub에 올려서 팀원들과 공유한다"가 정확한 표현입니다.

## 2. CLI vs GUI, 실무에서는 뭘 쓸까? ⌨️ vs 🖱️

Git을 설치하고 나면 두 가지 방법으로 Git을 다룰 수 있습니다.
- CLI (Command Line Interface): 검은 화면(터미널)에 git add, git commit처럼 직접 명령어를 치는 방식입니다.
- GUI (Graphical User Interface): 소스트리(Sourcetree), 깃크라켄(GitKraken), 혹은 VS Code 내장 기능처럼 마우스 클릭으로 조작하는 시각적인 방식입니다.

**"실무에서는 무조건 멋있게 터미널(CLI)로 명령어를 쳐야 진짜 개발자 아닌가요?"**
반은 맞고 반은 틀립니다! 실무에서는 두 가지를 섞어서 씁니다.
- 코드 수정 후 가볍게 저장(commit)하거나, 서버에 올릴(push) 때는 속도가 빠른 CLI(터미널)를 주로 씁니다.
- 하지만 코드가 복잡하게 꼬여서 전체적인 흐름(히스토리)을 봐야 하거나, 다른 사람의 코드와 내 코드를 병합(merge)하면서 충돌을 해결해야 할 때는 시각적으로 훌륭한 GUI 도구를 쓰는 것이 훨씬 효율적이고 안전합니다.

이 튜토리얼에서는 뼈대가 되는 CLI 명령어를 위주로 배우되, 실무에서 GUI가 빛을 발하는 순간들도 함께 짚어드릴 예정입니다.

## 3. 실무 투입 전, 내 이름표 달기 (초기 세팅) 🏷️

Git을 설치했다면 본격적으로 코드를 기록하기 전에 "내가 누구인지" Git에게 알려주어야 합니다. 그래야 나중에 "이 버그 누가 만들었어?!" 할 때 범인(?)을 찾을 수 있거든요.
터미널(또는 Git Bash)을 열고 아래 명령어들을 차례대로 입력해 주세요.

#### 1) 이름과 이메일 설정
```cli
git config --global user.name "여러분의 영문 이름"
git config --global user.email "여러분의 이메일 주소"
```

$$💡실무 꿀팁$$

```
이메일 주소는 신중하게!
여기서 입력하는 이메일 주소는 GitHub에 가입할 때 사용한 이메일 주소와 똑같이 맞추는 것이 좋습니다.
그래야 나중에 코드를 GitHub에 올렸을 때,
내 프로필에 영롱한 초록색 잔디(Contribution)가 예쁘게 심어집니다!
```

#### 2) 잘 설정되었는지 확인하기
```cli
git config --list
```
이 명령어를 쳤을 때 방금 설정한 `user.name`과 `user.email`이 보인다면 성공입니다! 빠져나오려면 키보드 q를 누르시면 됩니다.

#### 3) (선택) 기본 브랜치명 변경하기 (master -> main)

과거에는 Git의 기본 작업 공간(브랜치) 이름이 `master`였지만, 최근에는 IT 업계 전반에서 노예제를 연상시키는 단어를 퇴출하는 운동에 따라 `main`을 기본으로 사용합니다. GitHub도 `main`을 기본으로 씁니다.
아래 명령어를 쳐두면 앞으로 새로운 프로젝트를 시작할 때 기본 이름이 `main`으로 설정됩니다.
```cli
git config --global init.defaultBranch main
```

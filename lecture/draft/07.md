"앗, 지금 다른 브랜치로 가야해!": Stash 📦

안녕하세요! 지난 6편까지 우리는 코드를 저장하고, 브랜치를 나누고, 원격 저장소에서 협업하는(PR) 필수적인 과정들을 모두 마스터했습니다.

이제 여러분은 능숙하게 `feature/login` 브랜치에서 열심히 로그인 기능을 개발하고 있습니다. 코드를 한 절반쯤 짰을까요? 갑자기 사수님이 다급하게 달려옵니다.
```
👨‍💻 사수: "A씨!! 지금 라이브 서버 결제 쪽에 치명적인 에러가 났어요! 당장 main 브랜치로 가서 결제 버튼 좀 고쳐주세요!!"
```
자, 멘붕의 시작입니다.
로그인 기능은 아직 반밖에 완성 안 돼서 에러가 팍팍 나고 있기 때문에 `commit`을 하기엔 너무 지저분합니다. 그렇다고 이 상태로 `git switch main`을 치면, Git은 **"너 아직 작업 중인 거 저장 안 했잖아! 이동 못 해!"** 라며 딴지를 걸거나, 수정 중이던 코드를 `main` 브랜치까지 질질 끌고 가버립니다.

이럴 때 여러분을 구원해 줄 마법의 상자, Stash(스태시)를 소개합니다!

## 1. Stash란 무엇인가요? 🧳

'Stash'는 영어로 '안전한 곳에 넣어 두다, 숨겨 두다'라는 뜻입니다.

지금 내가 작업하고 있던 코드(수정사항)들을 찰흙 뭉치듯 뭉쳐서 **임시 보관 상자**에 잠시 던져두는 기능입니다. Stash를 쓰면 내 작업 공간(Working Directory)은 방금 커밋을 마친 것처럼 아주 깨끗한 상태로 돌아갑니다.

마음 편하게 다른 브랜치로 넘어가서 긴급 작업을 마친 뒤, 다시 돌아와서 보관 상자에 넣어둔 코드를 꺼내오면 되는 것이죠!

## 2. 🪄 Stash 실전 시나리오 한 바퀴

방금 전의 다급한 상황으로 돌아가서, 명령어를 통해 위기를 극복해 봅시다.

#### 📦 Step 1. 작업 중인 코드 임시 저장하기: `git stash`
```
# 변경사항을 임시 보관 상자에 넣습니다.
git stash
```

명령어를 치면 `Saved working directory and index state...`라는 메시지와 함께, 방금 전까지 에디터에 있던 어지러운 코드들이 싹! 사라집니다. (삭제된 게 아니라 안전하게 보관된 것이니 당황하지 마세요!)

#### 💡 **\[실무 꿀팁\] 이름표 붙여서 넣기**
실무에서는 상자에 뭘 넣었는지 기억하기 위해 메모를 적어두는 것이 좋습니다.
```
git stash push -m "로그인 기능 개발 중 (절반 완성)"
```

#### 🏃 Step 2. 긴급 작업 다녀오기

이제 작업 공간이 깨끗해졌으니 안전하게 브랜치를 이동할 수 있습니다.
```
git switch main           # 메인 브랜치로 이동
git switch -c hotfix/pay  # 긴급 수정용 브랜치 생성
# ... 결제 버튼 버그 수정 후 commit, PR 완료!
```

#### 📥 Step 3. 원래 하던 작업 꺼내오기: `git stash pop`
긴급 불을 껐으니, 다시 원래 하던 로그인 작업을 하러 돌아갈 시간입니다.
```
git switch feature/login  # 내 원래 작업 브랜치로 복귀

# 임시 보관 상자에서 코드 꺼내기!
git stash pop
```
`pop`이라는 단어에서 느껴지듯, 보관 상자에서 코드를 '뽕!' 하고 뽑아옵니다. 아까 사라졌던 코드가 마법처럼 다시 에디터에 나타나며, 보관 상자(Stash)는 다시 비워집니다.

## 3. 🔍 보관 상자 안이 궁금할 때 쓰는 명령어들
Stash는 여러 번 할 수 있어서 상자 안에 여러 개의 코드를 쌓아둘 수도 있습니다.

#### 📋 상자 리스트 확인하기: `git stash list`
```
git stash list
```

`stash@{0}: WIP on...`, `stash@{1}: On feature...`
이렇게 상자 안에 보관된 내역들을 목록으로 보여줍니다. (가장 최근에 넣은 것이 0번입니다.)

#### 📥 꺼내면서 상자 안에도 남겨두기: `git stash apply`
`pop`은 코드를 꺼내오면서 보관 상자를 비워버립니다. 하지만 혹시 몰라서 상자 안에도 복사본을 남겨두고 싶다면 `apply`를 씁니다.
```
git stash apply
```

#### 🗑️ 필요 없는 상자 버리기: `git stash drop`
`apply`를 써서 상자 안에 찌꺼기가 남았거나, 아예 필요 없는 임시 저장본을 삭제할 때 씁니다.
```
git stash drop       # 가장 최신 상자 버리기
git stash clear      # 상자 전체 다 비워버리기 (주의!)
```

## 🎯 오늘 배운 내용 요약
1. 아직 커밋하기 애매한데 다른 브랜치로 가야 할 때는 Stash를 쓰자!
2. 코드 임시 저장하기 👉 `git stash` (또는 `push -m "메시지"`)
3. 다른 작업 후 원래 브랜치로 돌아와서 코드 꺼내기 👉 `git stash pop`

`stash`는 정말 실무에서 숨 쉬듯이 쓰이는 구명조끼 같은 명령어입니다. 이 명령어만 알아도 갑작스러운 업무 지시가 두렵지 않을 거예요!

우리 팀은 어떻게 일할까? (Git Branching Strategies) 🏢

안녕하세요! 지난 11편까지 우리는 Git이라는 강력한 도구를 자유자재로 다루는 법을 배웠습니다. 이제 `rebase`도 하고, 실수로 날린 커밋도 살려낼 수 있는 여러분은 이미 Git 장인입니다!

하지만 실무의 현장은 명령어 실력만으로 해결되지 않습니다. 백엔드 개발자, 프론트엔드 개발자, QA 담당자가 한 프로젝트에서 매일 수십 번씩 코드를 합치는데, **"아무나 마음대로 `main`에 push 하면 서비스가 터지겠죠?"**

그래서 팀마다 "우리 팀은 이런 규칙으로 코드를 합치자!"라고 정해둔 약속이 있는데, 이것이 바로 브랜치 전략(Branching Strategies)입니다. 오늘은 현대 소프트웨어 개발에서 가장 많이 쓰이는 3가지 규칙을 소개합니다.

## 1. Git Flow: "교과서적인 완벽주의" 🧱
가장 전통적이고 유명한 방식입니다. 덩치가 크고 서비스가 복잡한 대형 프로젝트에서 주로 사용합니다. 아주 체계적이지만, 그만큼 복잡합니다.

- **브랜치 구조**:
  - `main`: 실제 서비스되는 최종 코드
  - `develop`: 다음 버전을 개발하는 기준 브랜치
  - `feature/*`: 기능 단위 개발 브랜치
  - `release/*`: 배포 직전 테스트 브랜치
  - `hotfix/*`: 긴급 버그 수정 브랜치

- **장점**: 배포 단위가 명확하고 프로세스가 완벽하게 갖춰져 있어 대규모 협업에 매우 안정적입니다.
- **단점**: 브랜치 개수가 너무 많아서 관리가 힘들고, 속도가 느립니다. "이 기능 언제 배포돼요?" 하면 브랜치 타고 넘어가느라 시간이 걸릴 수 있습니다.

## 2. GitHub Flow: "빠르고 가벼운 민첩함" 🚀
Git Flow가 너무 무겁고 느리다고 느껴질 때 등장한 대안입니다. 현대적인 웹 서비스, 특히 **CI/CD(지속적 통합/배포)** 환경에 아주 최적화되어 있습니다.

- **브랜치 구조**:
  - `main`: 언제든지 배포 가능한 상태의 코드
  - `feature/*`: 새로운 기능 개발 브랜치 (작업 끝나면 바로 main으로 Merge)
- **장점**: 매우 단순하고 빠릅니다. 기능 하나를 만들면 바로 테스트하고 배포하는 과정이 반복됩니다.
- **단점**: `main`브랜치가 항상 배포 가능한 상태여야 하므로, 팀원들의 테스트 자동화(자동화된 테스트 코드) 환경이 아주 잘 갖춰져 있어야 합니다. (초보 팀에게는 위험할 수도 있어요!)

## 3. Trunk-based Development: "DevOps의 끝판왕" ⚡
최근 구글, 페이스북 같은 빅테크 기업들이 가장 선호하는 방식입니다. 브랜치를 아예 거의 만들지 않거나, 아주 짧게만 유지하는 방식입니다.

- **브랜치 구조**:
  - `main`(Trunk): 모든 개발자가 여기서 바로 작업하거나, 아주 짧은 수명의 브랜치를 만들어 즉시 Merge 합니다.
- **장점**: 코드 병합 충돌이 일어날 틈이 없습니다. 매일 코드를 합치기 때문에 프로젝트가 항상 최신 상태를 유지하고 배포 속도가 극도로 빠릅니다.
- **단점**: 난이도가 최상입니다. 테스트 코드가 엄청나게 탄탄해야 하며, 실수로 버그 코드를 넣으면 서비스 전체가 멈출 수 있기 때문에 뛰어난 역량이 필요합니다.

#### 💡 그래서, 우리 팀은 뭘 써야 하죠?
- **팀 규모가 작고, 빠르게 결과물을 배포해야 한다면?**
  - 👉 **GitHub Flow**를 추천합니다. 가장 배우기 쉽고 실무 적응도 빠릅니다.
- **금융, 보안 등 실수가 용납되지 않는 대규모 시스템이라면?**
  - 👉 **Git Flow**의 엄격한 절차를 고민해 보세요.
- **우리 팀은 테스트 코드가 완벽하고, 배포 자동화가 잘 되어 있다!**
  - 👉 **Trunk-based**를 도전하세요!

## 🎯 오늘 배운 내용 요약
1. 브랜치 전략은 팀이 서로 안전하게 코드를 합치기 위한 '약속'이다.
2. **Git Flow**는 엄격하고 체계적(대규모), **GitHub Flow**는 빠르고 심플(중소규모), **Trunk-based**는 극강의 속도(고수팀).
3. 우리 팀이 어떤 전략을 쓰는지 파악하는 것이 곧 '실무 적응'의 첫걸음이다!

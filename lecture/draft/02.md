 실무자의 하루 1: 변경사항 저장하기 💾

안녕하세요! 지난 1편에서는 Git이 무엇인지 알아보고 내 이름표를 다는 초기 세팅까지 마쳤습니다.

오늘은 드디어 실전입니다. 회사에 출근해서 코드를 수정하고, 그 변경사항을 안전하게 Git에 기록하는 '가장 기본적이고 매일 반복되는 사이클'을 배워보겠습니다.

이 과정만 완벽히 이해해도 Git의 절반은 마스터했다고 볼 수 있습니다! 🚀

1. Git의 3가지 공간: "쇼핑을 하러 가볼까요?" 🛒

명령어를 치기 전에 딱 하나만 머릿속에 그려봅시다. Git은 파일의 상태를 관리하기 위해 크게 3가지 공간을 사용합니다. 마트에서 쇼핑하는 상황을 상상해 보세요!

Working Directory (작업 공간 = 마트 진열대):
여러분이 현재 코드를 작성하고 수정하는 실제 폴더입니다. 아직 계산대(Git)에 올리지 않은 상태입니다.

Staging Area (대기 공간 = 쇼핑 카트):
수정한 파일 중, "이번에 기록(결제)할 파일들만" 골라서 담아놓는 임시 공간입니다.

Repository (저장소 = 영수증/결제 완료):
쇼핑 카트에 담긴 파일들을 최종적으로 결제하고, 영원히 남는 영수증(버전)으로 기록한 상태입니다.

자, 이제 이 흐름대로 명령어를 입력해 볼까요?

2. 실전 명령어 한 바퀴: add, commit, status, log

어떤 프로젝트 폴더 안에서 git init 명령어를 쳐서 Git 관리를 시작했다고 가정해 봅시다. (또는 나중에 배울 git clone으로 프로젝트를 다운받은 상태여도 같습니다.)

📝 Step 1. 내 상태 확인하기: git status

Git을 쓸 때 습관처럼 쳐야 하는 명령어입니다. 현재 어떤 파일이 수정되었고, 어떤 파일이 카트(Staging Area)에 담겨 있는지 친절하게 알려줍니다.

git status



(빨간색 글씨로 수정된 파일 이름이 뜬다면, 아직 카트에 담기지 않았다는 뜻입니다.)

🛒 Step 2. 카트에 담기: git add

수정한 파일(login.js라고 해봅시다)을 저장하기 위해 카트에 담습니다.

# 특정 파일만 카트에 담기
git add login.js

# 수정된 모든 파일을 한 번에 카트에 담기 (실무에서 가장 많이 씀!)
git add .



git add . (점)을 입력하고 다시 git status를 쳐보면, 빨간색이었던 파일 이름이 초록색으로 변해 있을 겁니다. 카트에 잘 담겼다는 뜻이죠!

💳 Step 3. 결제하고 영수증 남기기: git commit

이제 카트에 담긴 파일들을 하나의 버전으로 묶어 기록합니다. 이때, 영수증에 "무엇을 샀는지(수정했는지)" 메모를 남겨야 합니다. 이것을 커밋 메시지라고 합니다.

git commit -m "로그인 버튼 색상 파란색으로 변경"



-m은 message의 약자입니다. 따옴표 안에 내가 무엇을 수정했는지 명확하게 적어주면 됩니다.

🧾 Step 4. 지난 영수증(기록) 확인하기: git log

내가 지금까지 결제한 영수증 목록, 즉 커밋 히스토리를 보고 싶을 때 사용합니다.

git log



누가, 언제, 어떤 커밋 메시지로 코드를 저장했는지 시간순으로 쫙 나옵니다. (빠져나오려면 키보드 q를 누르세요!)

💡 

$$실무 꿀팁$$

 "아무렇게나 적지 마세요!" 커밋 메시지 컨벤션

개인 프로젝트를 할 때는 커밋 메시지를 이렇게 적는 분들이 많습니다.

💬 "수정"
💬 "진짜최종"
💬 "아 왜 안돼"

하지만 실무에서 이렇게 적었다간 사수에게 등짝을 맞기 십상입니다. 😅 수많은 팀원들이 코드를 함께 보기 때문에, 규칙을 정해서 커밋 메시지를 작성하는 것이 매우 중요합니다.

현재 IT 업계에서 가장 널리 쓰이는 규칙(Conventional Commits) 몇 가지를 소개합니다.

feat : 새로운 기능 추가 (예: feat: 카카오 로그인 기능 추가)

fix : 버그 수정 (예: fix: 로그인 버튼 안 눌리는 현상 수정)

docs : 문서 수정 (예: docs: README.md 오타 수정)

style : 코드 포맷팅, 세미콜론 누락 수정 등 (코드 로직 변경 없음)

refactor : 코드 리팩토링 (기능은 똑같지만 코드를 깔끔하게 개선함)

chore : 빌드 업무 수정, 패키지 매니저 수정 등 (예: chore: 라이브러리 업데이트)

이렇게 말머리를 달아두면 git log를 볼 때 이 커밋이 기능을 추가한 건지, 버그를 고친 건지 한눈에 파악할 수 있어 협업 효율이 수직 상승합니다!

🎯 오늘 배운 내용 요약

현재 상태 파악은 수시로! 👉 git status

수정한 파일을 카트(Staging Area)에 담기 👉 git add .

명확한 메시지(말머리 활용!)와 함께 기록 남기 👉 git commit -m "메시지"

지금까지의 기록 확인하기 👉 git log

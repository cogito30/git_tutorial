세상으로 나가는 포트폴리오: GitHub Pages 🌍

안녕하세요! 그동안 브랜치도 나누고, PR도 올리며 '실무형 개발자'의 면모를 갖춰온 여러분, 정말 고생 많으셨습니다.

이제 로컬 폴더 속에만 갇혀 있던 우리의 포트폴리오를, 전 세계 누구나 접속할 수 있는 '살아있는 사이트'로 탈바꿈시켜 볼 시간입니다. 서버를 빌리고, 도메인을 사고... 벌써 머리가 아프신가요? 걱정 마세요. GitHub Pages를 사용하면 클릭 몇 번으로 무료 배포가 가능합니다.

## 1. GitHub Pages가 도대체 뭔가요? 🤔
GitHub Pages는 GitHub에서 제공하는 **정적 웹 호스팅 서비스**입니다.

- **공짜입니다**: 학생이나 취업 준비생에게 서버 비용은 부담이죠? GitHub Pages는 완전히 무료입니다.
- **배포가 쉽습니다**: 따로 서버에 접속해서 파일을 업로드할 필요가 없습니다. 우리가 지금까지 하던 대로 `git push`만 하면 알아서 사이트가 업데이트됩니다.
- **포트폴리오에 딱입니다**: HTML/CSS/JS로 만든 정적 페이지는 물론, React, Next.js 등 최신 프론트엔드 프레임워크로 만든 사이트도 다 배포할 수 있습니다.

## 2. 배포의 마법 시작하기 🪄
자, 이제 터미널을 끄고 GitHub 사이트로 접속해 봅시다.

#### 1단계: 저장소 설정 들어가기
GitHub 내 포트폴리오 저장소(`my-portfolio`) 페이지로 이동한 뒤, 상단 탭에서 `Settings`을 클릭하세요.

#### 2단계: Pages 메뉴 활성화

왼쪽 메뉴바에서 `Pages` 탭을 클릭합니다. 여기서 우리가 사이트 파일이 있는 브랜치를 알려주어야 합니다.

1. **Build and deployment** 섹션 확인.
2. Source를 `Deploy from a branch`로 선택.
3. Branch 항목에서 `main`(또는 `gh-pages`) 브랜치를 선택하고 `/(root)`경로를 선택한 뒤 `Save` 버튼을 누르세요.

#### 3단계: 주소 확인하기
약 1~2분 정도 기다리면, 상단에 `Your site is live at https://여러분의아이디.github.io/my-portfolio/` 라는 문구가 뜹니다.

**축하합니다! 이제 여러분의 포트폴리오가 전 세계 어디서든 접속 가능한 링크를 갖게 되었습니다!**

## 3. 이제 '지속적인 배포'를 즐기세요 🚀
이제부터는 여러분의 작업 루틴이 완전히 바뀝니다.
1. 로컬에서 기능을 수정한다.
2. `git add` & `git commit` & `git push`를 한다.
3. 1분 뒤, GitHub Pages 주소에 접속하면 **내가 수정한 내용이 즉시 반영되어 있다**.

이것이 바로 개발자들이 말하는 '지속적 배포(CD)'의 맛보기입니다. 이제부터 여러분의 포트폴리오는 '완성된 결과물'이 아니라, 매일매일 조금씩 더 나은 모습으로 발전하는 '현재 진행형 서비스'가 되었습니다.

## 🎯 오늘 배운 내용 요약
1. **GitHub Pages**는 포트폴리오 사이트를 무료로 배포할 수 있는 최고의 도구다.
2. 설정(Settings) -> Pages에서 브랜치만 지정하면 배포 끝!
3. `git push`만 하면 사이트가 업데이트되는 '지속적 배포'의 짜릿함을 경험하자.

이제 여러분의 결과물이 웹에서 누구나 접속할 수 있는 사이트로 거듭났습니다. 주변 친구들에게 링크를 보내 자랑해 보세요!

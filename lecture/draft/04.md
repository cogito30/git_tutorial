$$실무 Git 튜토리얼 4편$$

 두 세계의 만남: Merge와 충돌(Conflict) 해결 💥

안녕하세요! 지난 3편에서는 내 작업 공간을 안전하게 분리하는 '브랜치(Branch)'라는 평행우주를 만들어 보았습니다.

자, feature/payment(결제 기능) 브랜치에서 밤을 새워가며 완벽한 코드를 작성했다고 가정해 봅시다. 이제 이 멋진 기능을 실제 서비스가 돌아갈 원본 코드, 즉 main 우주로 가져와서 합쳐야겠죠?

오늘은 두 세계를 하나로 합치는 Merge(병합)와, 모든 주니어 개발자의 심장을 철렁하게 만드는 Merge Conflict(충돌) 현상을 극복하는 방법을 알아보겠습니다.

1. 브랜치 합치기: git merge 🤝

브랜치를 합칠 때 가장 중요한 절대 규칙이 하나 있습니다.
바로 "내가 코드를 가져올 목적지(Base) 브랜치로 먼저 이동해야 한다"는 것입니다.

feature/payment 브랜치의 코드를 main으로 합치고 싶다면, 내비게이션(HEAD)이 main을 가리키고 있어야 합니다.

🌿 Step 1. 목적지 브랜치로 이동하기

git switch main



🧲 Step 2. 코드 빨아들이기 (Merge)

이제 main 브랜치에 선 상태에서, 합치고 싶은 브랜치 이름을 부릅니다.

git merge feature/payment



"내(main)가 있는 곳으로 feature/payment 의 변경사항을 가져와서 합쳐줘!"라는 뜻입니다. 아무 문제가 없다면 촤르륵~ 글씨가 올라가며 두 브랜치의 코드가 평화롭게 하나로 합쳐집니다.

2. 공포의 빨간 글씨, 충돌(Merge Conflict) 발생! 🚨

하지만 실무에서는 평화로운 병합만 일어나지 않습니다. A 개발자와 B 개발자가 동시에 작업하다가 "같은 파일의 같은 줄"을 수정해버린다면 어떻게 될까요?

Git은 멘붕에 빠집니다. "어? main 브랜치에서는 이 줄이 '파란색'인데, 방금 가져온 브랜치에서는 '빨간색'이네? 내가 임의로 하나를 지울 수는 없는데... 주인이 직접 결정해 줘!"

이때 터미널에 무시무시한 에러 메시지가 뜹니다.

CONFLICT (content): Merge conflict in index.html
Automatic merge failed; fix conflicts and then commit the result.



이것이 바로 Merge Conflict(병합 충돌)입니다. 절대 당황하지 마세요. 에러가 아니라, Git이 "나 혼자 결정 못 하겠으니 도와주세요"라고 SOS를 친 것뿐입니다.

3. 우아하게 충돌 해결하기 (feat. VS Code) 🪄

충돌이 난 파일(index.html)을 열어보면 외계어 같은 기호들이 추가되어 있습니다.

<<<<<<< HEAD
<button class="btn-blue">결제하기</button>
=======
<button class="btn-red">결제하기</button>
>>>>>>> feature/payment



<<<<<<< HEAD 부터 ======= 까지: 현재 내가 있는 브랜치(main)의 코드

======= 부터 >>>>>>> 브랜치명: 합치려고 가져온 브랜치의 코드

💡 

$$실무 꿀팁$$

 터미널에서 지우지 말고, 에디터를 쓰세요!

과거에는 저 꺾쇠 기호(<<<, ===, >>>)를 백스페이스로 일일이 지우고 코드를 수정했습니다. 하지만 실무에서는 VS Code 같은 에디터의 내장 기능을 활용하여 1초 만에 해결합니다.

VS Code에서 충돌 난 파일을 열면, 충돌 난 부분 위에 다음과 같은 클릭 가능한 버튼 4개가 나타납니다.

Accept Current Change (현재 변경 사항 수락): 기존 main 코드를 유지할래!

Accept Incoming Change (수신 변경 사항 수락): 새로 가져온 브랜치 코드로 덮어쓸래!

Accept Both Changes (두 변경 사항 모두 수락): 둘 다 살릴래! (코드가 위아래로 나란히 남습니다)

Compare Changes (변경 사항 비교): 두 코드를 양옆으로 띄워놓고 자세히 볼래!

상황에 맞게 버튼을 딱! 클릭하면 지저분한 기호들이 싹 사라지며 코드가 깔끔하게 정리됩니다. (직접 코드를 타이핑해서 두 코드를 적절히 섞어 새로운 코드를 만들어도 됩니다.)

✅ Step 3. 해결 완료 후 다시 커밋하기

에디터에서 코드를 입맛에 맞게 고쳤다면, 변경된 파일을 저장하고 Git에게 "충돌 해결 끝났어!"라고 알려주어야 합니다. 평소에 하던 저장(add, commit)과 똑같습니다.

git add index.html
git commit -m "fix: index.html 결제 버튼 충돌 해결"



축하합니다! 가장 큰 고비인 충돌을 무사히 넘기고 병합을 완료했습니다. 🎉

🎯 오늘 배운 내용 요약

브랜치를 합칠 때는 목적지 브랜치(main)로 먼저 이동(switch)해야 한다.

합치기 명령어 👉 git merge 합칠브랜치명

같은 곳을 수정해서 충돌(Conflict)이 났다면 당황하지 말고 VS Code 버튼(Accept ~ Change)을 클릭해 해결한다.

충돌 해결 후에는 반드시 다시 add와 commit을 해준다.

여기까지 따라오셨다면, 내 컴퓨터(Local) 안에서 일어나는 Git의 핵심적인 작업은 모두 마스터하신 겁니다!

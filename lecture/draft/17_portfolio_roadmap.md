$$\[기획\] Git으로 완성하는 나만의 포트폴리오 사이트 제작기 🚀$$

**기획 의도**: 단순히 "웹사이트를 하나 만들었다"에서 끝나지 않고, 기획부터 설계, 배포, 유지보수까지의 전 과정을 현업 표준 Git 워크플로우에 맞춰 진행합니다. 이를 통해 채용 담당자에게 "팀 협업과 프로세스를 완벽히 이해하고 있는 준비된 신입 엔지니어"임을 증명하는 최고의 포트폴리오를 구축합니다.

## 🟢 Part 1. 기획과 기초 설계 (Foundation)

#### \[1편\] 왜 Git으로 포트폴리오를 만드나요?
- **핵심**: 포트폴리오에서 Git 히스토리가 중요한 이유, 나에게 맞는 기술 스택 선정, GitHub 원격 저장소 초기화 및 첫 커밋 생성

#### \[2편\] 프로의 기본기: .gitignore와 커밋 컨벤션
- **핵심**: 보안과 프로젝트 경량화를 위한 .gitignore 작성 규칙, 실무 수준의 일관된 커밋 규칙(Conventional Commits) 수립 및 적용

## 🟡 Part 2. 브랜치 전략과 협업 흐름 (Branching & PR)

#### \[3편\] 기능별로 쪼개기: Feature Branch Flow
- **핵심**: 원본 코드를 보존하며 안전하게 작업하는 브랜치 분할 전략, 실무 협업 표준 브랜치 규칙 설정

#### \[4편\] 나 자신과 협업하기: PR과 셀프 코드 리뷰 (Self-Review)

- **핵심**: 혼자 작업할 때도 Pull Request(PR)를 작성해야 하는 이유, 스스로 작성한 코드를 객관적으로 검증하는 셀프 리뷰 루틴

## 🟠 Part 3. 배포 및 자동화 (Deployment & CI/CD)

#### \[5편\] 전 세계에 주소 공유하기: GitHub Pages 무료 웹 호스팅

- **핵심**: 정적 웹 호스팅의 원리 이해, 도메인/서버 비용 없이 GitHub을 통해 실시간 웹사이트 무료 배포하기

#### \[6편\] 배포 자동화: GitHub Actions로 CI/CD 구축하기

- **핵심**: 지속적 통합/배포(CI/CD)의 개념 파악, 코드 푸시 한 번으로 빌드부터 배포까지 스스로 동작하는 파이프라인 자동화

## 🟠 Part 4. 히스토리 관리와 코드 품질 보장 (Advanced)

#### \[7편\] 과거의 나를 세탁하기: Interactive Rebase와 Squash

- **핵심**: 개발 과정 중 발생한 자잘하고 지저분한 실수 커밋들을 rebase -i를 사용해 의미 있는 기능 단위로 예쁘게 병합하기

#### \[8편\] 자동화 경찰관 배치: Husky와 Linter 연동

- **핵심**: 협업 시 코드 일관성을 강제하는 Git Hooks의 이해, Husky를 사용해 빌드가 불가능하거나 컨벤션이 깨진 코드의 커밋/푸시 원천 차단하기

## 🟣 Part 5. 에필로그

#### \[9편\] Git 히스토리를 이력서로 승화시키는 법

- **핵심**: 매력적인 README.md 작성법, 채용 담당자에게 내 GitHub 저장소의 PR 히스토리와 커밋 로그를 효과적으로 어필하는 이력서 작성 팁

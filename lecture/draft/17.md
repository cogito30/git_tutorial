[기획] Git으로 완성하는 나만의 포트폴리오 사이트 제작기 🚀

기획 의도: Git을 배웠으니 이제 써먹어야겠죠? 포트폴리오 사이트를 기획부터 배포, 운영까지 전 과정을 Git 기반으로 진행하며 '실무형 개발자'임을 증명해 봅시다.

🟢 Part 1. 기획과 시작 (Foundation)

[1편] 왜 Git으로 포트폴리오를 만드나요?

주제: Git 활용이 포트폴리오의 퀄리티를 어떻게 높이는지(버전 관리, 히스토리)

핵심: 기술 스택 선정(HTML/CSS/JS 또는 정적 사이트 생성기), GitHub 저장소 초기화

[2편] Git으로 관리하는 첫 발걸음

주제: .gitignore 제대로 설정하기

핵심: 환경 변수, 빌드 파일 등 보안/용량 관리, 커밋 컨벤션(Conventional Commits) 설정

🟡 Part 2. 브랜치 전략 실습 (Branching Strategy)

[3편] 기능별로 쪼개기: Feature Branch Flow

주제: feature/about, feature/projects, feature/contact 등 기능별 브랜치 운용

핵심: 앞서 배운 switch -c와 병합 전략 적용하기

[4편] PR로 나 자신과 협업하기

주제: 본인의 main 브랜치에 Merge 할 때도 PR을 열어보자

핵심: 스스로 코드 리뷰를 진행하며 코드 품질 유지하기 (Self-Review 루틴 만들기)

🟠 Part 3. 배포와 자동화 (Deployment & CI/CD)

[5편] 세상으로 나가는 포트폴리오: GitHub Pages

주제: 별도 서버 없이 GitHub을 활용한 무료 배포

핵심: git push만 하면 사이트가 업데이트되는 마법 체험하기

[6편] 배포를 자동으로: GitHub Actions 도입

주제: Git 실무의 꽃, CI/CD 맛보기

핵심: Push 할 때마다 자동으로 사이트 빌드 및 배포하기

🔴 Part 4. 유지보수와 업그레이드 (Optimization)

[7편] 과거의 나를 탓하지 않기: 리팩토링과 Rebase

주제: 포트폴리오 코드를 수정하며 쌓인 지저분한 커밋 정리하기

핵심: rebase -i를 이용해 '아름다운 커밋 히스토리' 남기기

[8편] 자동화 경찰관: Husky와 Linter 적용

주제: 내 포트폴리오 사이트의 코드 품질을 유지하는 법

핵심: Prettier와 ESLint를 Git Hooks로 강제 적용하여 항상 깔끔한 코드 유지하기

🟣 Part 5. 에필로그

[9편] Git으로 증명하는 나의 성장

주제: 완성된 포트폴리오 저장소의 링크를 채용 담당자에게 어필하는 방법

핵심: README.md를 활용한 프로젝트 설명서 작성법 (Git 히스토리가 곧 포트폴리오가 되는 과정)

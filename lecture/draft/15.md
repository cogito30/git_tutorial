 Git 실무 마스터 과제 풀이 🔑

수고하셨습니다! 여러분이 시도한 방법과 비교해 보세요. 정답은 하나가 아니지만, 가장 권장되는 실무 흐름입니다.

1단계: 브랜치 생성

git switch -c feature/payment



2단계: 작업 및 커밋

# payment.js 수정 후
git add payment.js
git commit -m "feat: 결제 로직 구현"



3~4단계: 위기 대처 및 대피 (Stash 활용)

git stash             # 작업 내용을 임시 저장
git switch main       # main으로 복귀



5단계: 핫픽스 처리

git switch -c hotfix/typo
# index.html 수정 후
git add index.html
git commit -m "fix: 메인 페이지 오타 수정"
git switch main
git merge hotfix/typo
git branch -d hotfix/typo  # 완료된 브랜치 삭제



6단계: 작업 복구

git switch feature/payment
git stash pop         # 저장했던 작업 내용 가져오기



7단계: 깔끔하게 마무리 (Squash & Merge)

# feature/payment 브랜치에서 진행
git rebase -i main    # 최신 main 코드를 베이스로 가져오고, 커밋을 뭉치기 (Squash)
# 에디터에서 pick을 s(squash)로 변경하여 커밋 합침

git switch main
git merge feature/payment
git branch -d feature/payment



🎉 미션 완료!
이 흐름을 자연스럽게 수행할 수 있다면 여러분은 이미 실무에서 Git 때문에 당황할 일 없는 훌륭한 개발자입니다.

또 다른 시나리오나 더 궁금한 기능이 있다면 언제든 말씀해 주세요!

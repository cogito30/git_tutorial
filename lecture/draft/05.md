내 코드를 세상으로: Remote Repository ☁️

안녕하세요! 지난 4편까지 우리는 내 컴퓨터 안에서 브랜치를 만들고, 합치고, 충돌까지 해결하며 지지고 볶는(?) 모든 과정을 마스터했습니다.

하지만 실무에서 코드를 내 노트북에만 꽁꽁 숨겨둔다면 어떻게 될까요?
갑자기 노트북에 커피를 쏟아서 고장 나거나, 다른 팀원과 협업해야 할 때 코드를 USB에 담아서 건네줄 수는 없는 노릇입니다.

그래서 우리는 코드를 인터넷 공간(클라우드)에 올려서 안전하게 백업하고 팀원들과 공유해야 합니다. 오늘 배울 내용이 바로 Git의 단짝 친구, GitHub(원격 저장소)을 활용하는 방법입니다!

## 1. Local과 Remote: "내 일기장과 공개 게시판" 📓 vs 📋

Git의 세계에는 두 가지 종류의 저장소(Repository)가 있습니다.
- **Local Repository (로컬 저장소)**: 지금까지 우리가 작업했던 내 컴퓨터 안의 작업 공간입니다. (나만 볼 수 있는 개인 일기장)
- **Remote Repository (원격 저장소)**: GitHub, GitLab 같은 인터넷 서버에 있는 작업 공간입니다. (모두가 볼 수 있는 클라우드나 공개 게시판)

우리의 목표는 내 컴퓨터(Local)에 있는 버전을 인터넷(Remote)으로 복사해서 올려놓는 것입니다.

## 2. 내 코드 인터넷에 올리기: `remote` 와 `push` 🚀

GitHub에 새 Repository를 만들었다고 가정해 봅시다. 텅 빈 인터넷 공간이 생겼습니다. 이제 내 컴퓨터의 코드들을 이 공간으로 밀어 올려야 합니다.

#### 🔗 Step 1. 연결고리 만들기: `git remote add`
내 컴퓨터의 Git은 아직 인터넷 어디로 코드를 보내야 할지 모릅니다. 목적지 주소를 알려주어야 합니다.
```
git remote add origin https://github.com/내아이디/내저장소.git
```
- `remote add`: 원격 저장소를 추가하겠다!
- `origin`: 주소가 너무 기니까, 앞으로 이 주소를 **'origin'이라는 별명**으로 부르겠다! (실무에서는 거의 100% 원격 저장소의 기본 이름을 origin으로 씁니다.)
```
# 연결이 잘 되었는지 확인하려면?
git remote -v
```

#### 📤 Step 2. 코드 밀어 올리기: `git push`
목적지를 알았으니, 이제 내 코드를 발사해 봅시다!
```
git push origin main
```
"내 로컬의 `main` 브랜치에 있는 코드를, 저기 인터넷에 있는 `origin`이라는 공간의 `main` 브랜치로 밀어 올려(push) 줘!"라는 뜻입니다. 성공적으로 명령어가 실행되면, GitHub 사이트에서 내 코드가 예쁘게 올라가 있는 것을 확인할 수 있습니다.

## 3. 남의 코드(팀 프로젝트) 가져오기: `git clone` 📥
반대로 생각해서, 여러분이 회사에 처음 입사했다고 가정해 봅시다.
사수님이 "GitHub 주소 메신저로 드렸으니, 코드 다운받아서 세팅해 주세요~"라고 합니다. 이때 사용하는 마법의 명령어가 바로 `clone`입니다.
```
git clone https://github.com/회사아이디/프로젝트.git
```
`clone`(복제)이라는 이름답게, 인터넷에 있는 코드는 물론이고 **지금까지의 모든 커밋 히스토리(버전 기록)까지 내 컴퓨터로 통째로 복사해 옵니다.**

$$🚨 초보자 주의사항$$

 clone을 할 때는 git init을 하지 마세요!
이미 다른 사람이 git init을 해서 만들어둔 저장소를 통째로 다운받는 것이기 때문에, 빈 폴더에서 바로 clone만 하시면 됩니다.

변경된 최신 코드 받아오기: `pull` vs `fetch`

내가 집에 간 사이, 동료가 밤새 코드를 수정해서 GitHub에 올려두었습니다(`push`). 다음 날 출근한 나는 그 최신 코드를 내 컴퓨터로 내려받아야 작업을 이어갈 수 있겠죠?

이때 인터넷에서 코드를 내려받는 명령어는 두 가지가 있습니다. 이 둘의 차이를 아는 것이 실무에서 정말 중요합니다.

#### 1) 무지성 다운로드 + 병합: `git pull`
```
git pull origin main
```
원격 저장소의 최신 코드를 다운받은 뒤, **내 현재 코드와 즉시 합쳐버립니다(Merge)**.
가장 편하고 많이 쓰지만, 내가 작업하던 코드와 겹치는 부분이 있다면 갑자기 충돌(Conflict)이 뽝! 발생하며 아침부터 멘붕에 빠질 수 있습니다.

#### 2) 일단 택배 상자 내용물만 확인: `git fetch`
```
git fetch origin
```
원격 저장소의 최신 코드를 다운로드만 하고, 내 코드와 합치지는 않습니다.
"어떤 파일이 업데이트되었지?" 하고 먼저 내용을 확인해 볼 때 사용합니다. 확인 후 안전하다고 판단되면 그때 따로 `git merge`를 해서 내 코드와 합칩니다.


#### 👨‍💻 실무 가이드
- 내가 작업한 내용이 별로 없을 때는 편하게 `git pull`을 사용하세요. 하지만 내가 수정한 코드가 많아서 남의 코드와 겹칠까 봐 불안하다면, `git fetch`로 먼저 변경 사항을 눈으로 확인하는 것이 안전합니다!

## 🎯 오늘 배운 내용 요약
1. **Local vs Remote**: 내 컴퓨터(Local)와 인터넷(Remote)은 별개의 공간이다.
2. 원격 저장소 연결하기 👉 `git remote add origin 주소`
3. 내 코드 밀어 올리기 👉 `git push origin 브랜치명`
4. 프로젝트 통째로 다운받기 👉 `git clone 주소`
5. 최신 업데이트 받기 👉 바로 합치려면 `pull`, 내용만 확인하려면 `fetch`

이제 여러분은 다른 사람과 코드를 주고받을 수 있는 모든 준비를 마쳤습니다! 🎉

그런데 실무에서는 여러 명이 동시에 같은 프로젝트를 건드립니다. "내가 짠 코드 바로 main에 push 할게!" 하고 맘대로 코드를 섞어버리면 서비스가 망가질 수도 있겠죠?
